# Потенциальные улучшения

## Масштабирование

- База данных: Мигрировать с Firebase Realtime Database на Firestore или PostgreSQL для сложных запросов.
- Redis: Использовать Redis для общего состояния между несколькими экземплярами бота.

## Безопасность

- Секреты: Убедиться, что все API ключи зашифрованы и регулярно ротируются.
- Проверка ввода: Усилить проверку входных данных модальных окон и аргументов команд.
- Ограничение скорости: Реализовать более строгие глобальные ограничения скорости для предотвращения банов API.

## Оптимизация памяти

- Ограничения кэша: Применить более строгие ограничения на кэши взаимодействий и embed.
- Управление потоками: Убедиться, что аудио ресурсы уничтожаются сразу после использования.
- Слушатели событий: Аудит слушателей событий для предотвращения утечек памяти.

## Рефакторинг кода

- Внедрение зависимостей: Заменить глобальные переменные на внедренные зависимости для лучшей тестируемости.
- TypeScript: Мигрировать на TypeScript для безопасности типов.
- Модульные тесты: Добавить тесты Jest для утилитных функций и логики состояния.
