# Логирование и мониторинг

## Стратегия

- Асинхронная очередь: Предотвращает блокировку I/O во время высокого трафика.
- Архивирование: Старые логи сжимаются и хранятся в storage/logs/archive.
- Очистка: Логи старше 60 дней автоматически удаляются.
- Отслеживание ошибок: Необработанные исключения и отклонения обещаний логируются с трассировкой стека.

## Мониторинг

- Проверка здоровья: HTTP сервер предоставляет конечные точки /health и /radio-health.
- Управление памятью: Периодические проверки запускают сборку мусора, если использование памяти превышает пороги.
- Статистика: Статистика использования бота (серверы, команды, зикры) отслеживается и сохраняется в Firebase.

## Уровни логов

- debug: Подробная внутренняя информация.
- info: Общие операционные сообщения.
- warn: Потенциальные проблемы.
- error: Критические сбои.
- fatal: Ошибки, останавливающие систему.
