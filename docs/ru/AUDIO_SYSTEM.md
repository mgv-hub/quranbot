# Аудиосистема

## Движок воспроизведения

Использует @discordjs/voice для обработки аудио.

- Ресурсы: Создаются из HTTP потоков.
- Плеер: Управляет состоянием воспроизведения и ошибками.
- Подключение: Обрабатывает связь голосового шлюза.

## Обработка потоков

- Проверка: URL проверяются перед воспроизведением.
- Логика повторных попыток: Неудачные потоки повторяются с резервными вариантами.
- Длительность: Оценивается из заголовков или номера суры.

## Данные чтецов

- Источники: mp3quran.net и пользовательские JSON репозитории.
- Кэширование: Данные кэшируются в памяти и Firebase.
- Резервный вариант: Локальные файлы используются, если удаленные источники не работают.

## Радиосистема

- Проверка здоровья: Потоки периодически мониторятся.
- Переключение: Автоматическое переключение на работающие потоки.
- Постраничность: Радио перечислены страницами по 25.
