# Управление состоянием

## Состояние сервера

Каждый сервер имеет объект состояния времени выполнения, хранящийся в памяти.

- Подключение: Объект голосового подключения.
- Воспроизведение: Текущая сура, чтец и режим.
- Таймеры: Таймеры зикров и неактивности.
- Конфигурация: Режим управления и ID каналов.

## Постоянное состояние

Состояние синхронизируется с Firebase Realtime Database.

- Восстановление: Состояния восстанавливаются при перезапуске бота.
- Резервное копирование: Автоматическое резервное копирование каждые 5 минут.
- Очистка: Устаревшие данные удаляются для покинутых серверов.

## Восстановление состояния

1. Загрузить состояния из Firebase.
2. Проверить существование канала.
3. Переподключить голосовые каналы.
4. Возобновить воспроизведение, если применимо.

## Управление памятью

- Сборка мусора запускается при высоком использовании памяти.
- Кэш взаимодействий периодически очищается.
- Уничтоженные подключения автоматически очищаются.
