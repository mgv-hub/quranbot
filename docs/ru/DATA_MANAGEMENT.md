# Управление данными

## Получение

Расположено в core/data/.

- Источники: Данные получаются из mp3quran.net, aladhan.com и пользовательских JSON репозиториев.
- Методы: Использует node-fetch с пользовательскими заголовками и таймаутами.
- Проверка: Структуры данных проверяются перед загрузкой в глобальное состояние.

## Кэширование

- Кэш времени выполнения: Данные хранятся в глобальных переменных для быстрого доступа.
- Кэш Firebase: Критические данные резервируются в Firebase для постоянства.
- Локальный кэш: JSON файлы используются как резервный вариант, если удаленные источники не работают.

## Хранение

- Firebase: Используется для настроек сервера, состояний, ID управления и webhook пользователей.
- Локальное: Файлы резервных копий сжимаются и хранятся локально перед отправкой на каналы Discord.
- Окружение: Конфиденциальная конфигурация загружается через пользовательский загрузчик Envira.
