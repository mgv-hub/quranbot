# Архитектура и структура

## Философия дизайна

Проект следует модульной монолитной архитектуре. Логика разделена на отдельные домены, включая State, Data, Interactions и Utils для облегчения обслуживания. Использует пользовательские псевдонимы путей для управления глубокими структурами каталогов.

## Иерархия папок

- core/bot/: Точка входа и инициализация основного клиента.
- core/startup/: Логика загрузки и регистрация команд.
- core/state/: Управление состоянием и постоянство.
- core/interactions/: Обработка кнопок, меню и команд.
- core/data/: Получение данных и кэширование.
- core/utils/: Общие утилиты, включая логирование и Firebase.
- core/ui/: Построители embed и создатели компонентов.
- core/package/Envira/: Пользовательский загрузчик переменных окружения.

## Основные компоненты

- Клиент: Экземпляр Discord.js Client, управляемый в core/bot/core.js.
- Менеджер состояния: GuildStateManager и PersistentStateManager обрабатывают состояние времени выполнения и базы данных.
- Процессор взаимодействий: Направляет все взаимодействия Discord к конкретным обработчикам.
- Загрузчик данных: Управляет внешними вызовами API и кэшированием для чтецов и сур.

## Взаимодействие модулей

Последовательность запуска инициализирует клиент, загружает данные, подключается к Firebase и восстанавливает предыдущие состояния. Взаимодействия направляются через центральный процессор, который проверяет разрешения и задержки перед выполнением конкретных обработчиков.
