# डेटा मैनेजमेंट

## फेचिंग

core/data/ में स्थित।

- स्रोत: डेटा mp3quran.net, aladhan.com और कस्टम JSON रिपॉजिटरी से फेच किया जाता है।
- विधियां: कस्टम हेडर और टाइमआउट के साथ node-fetch का उपयोग करता है।
- सत्यापन: डेटा संरचनाएं ग्लोबल स्टेट में लोड होने से पहले सत्यापित की जाती हैं।

## कैशिंग

- रनटाइम कैश: तेज़ एक्सेस के लिए डेटा ग्लोबल वेरिएबल्स में संग्रहीत होता है।
- Firebase कैश: महत्वपूर्ण डेटा परसिस्टेंस के लिए Firebase में बैकअप किया जाता है।
- स्थानीय कैश: यदि रिमोट स्रोत विफल होते हैं तो JSON फाइलें बैकअप के रूप में उपयोग की जाती हैं।

## स्टोरेज

- Firebase: गिल्ड सेटअप, स्टेट, नियंत्रण ID और उपयोगकर्ता webhook के लिए उपयोग किया जाता है।
- स्थानीय: बैकअप फाइलें डिस्कॉर्ड चैनल को भेजने से पहले संपीड़ित और स्थानीय रूप से संग्रहीत होती हैं।
- एनवायरनमेंट: संवेदनशील कॉन्फ़िगरेशन कस्टम Envira लोडर के माध्यम से लोड किया जाता है।
