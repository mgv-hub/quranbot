# ऑडियो सिस्टम

## प्लेबैक इंजन

ऑडियो हैंडलिंग के लिए @discordjs/voice का उपयोग करता है।

- संसाधन: HTTP स्ट्रीम से बनाए गए।
- प्लेयर: प्लेबैक स्टेट और त्रुटियों को प्रबंधित करता है।
- कनेक्शन: वॉइस गेटवे संचार को संभालता है।

## स्ट्रीम हैंडलिंग

- सत्यापन: प्लेबैक से पहले URL जांचे जाते हैं।
- पुनः प्रयास लॉजिक: विफल स्ट्रीम को बैकअप के साथ पुनः प्रयास किया जाता है।
- अवधि: हेडर या सूरह नंबर से अनुमानित।

## क़ारी डेटा

- स्रोत: mp3quran.net और कस्टम JSON रिपॉजिटरी।
- कैशिंग: डेटा मेमोरी और Firebase में कैश किया जाता है।
- बैकअप: यदि रिमोट स्रोत विफल होते हैं तो स्थानीय फाइलें उपयोग की जाती हैं।

## रेडियो सिस्टम

- स्वास्थ्य जांच: स्ट्रीम आवधिक रूप से मॉनिटर किए जाते हैं।
- फेलओवर: कार्यशील स्ट्रीम में स्वचालित स्विच।
- पेजिनेशन: रेडियो 25 के पेज में सूचीबद्ध।
