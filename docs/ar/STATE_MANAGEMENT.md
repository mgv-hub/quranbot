# إدارة الحالة

## حالة السيرفر

كل سيرفر لديه كائن حالة وقت تشغيل مخزن في الذاكرة.

- الاتصال: كائن الاتصال الصوتي.
- التشغيل: السورة الحالية، والقارئ، والوضع.
- المؤقتات: مؤقتات الأذكار وعدم النشاط.
- التكوين: وضع التحكم ومعرفات القنوات.

## الحالة المستمرة

تتم مزامنة الحالة مع Firebase Realtime Database.

- الاستعادة: يتم استعادة الحالات عند إعادة تشغيل البوت.
- النسخ الاحتياطي: نسخ احتياطية تلقائية كل 5 دقائق.
- التنظيف: إزالة البيانات القديمة للسيرفرات المغادرة.

## استعادة الحالة

1. تحميل الحالات من Firebase.
2. التحقق من وجود القنوات.
3. إعادة الاتصال بالقنوات الصوتية.
4. استئناف التشغيل إن أمكن.

## إدارة الذاكرة

- جمع القمامة يُطلق عند الاستخدام العالي للذاكرة.
- ذاكرة التخزين المؤقت للتفاعلات تُمسح دورياً.
- الاتصالات المدمرة تُنظف تلقائياً.
