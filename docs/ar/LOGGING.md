# التسجيل والمراقبة

## الاستراتيجية

- قائمة الانتظار غير المتزامنة: تمنع حظر I/O أثناء الحركة العالية.
- الأرشفة: السجلات القديمة تُضغط وتُخزن في storage/logs/archive.
- التنظيف: السجلات الأقدم من 60 يوماً تُحذف تلقائياً.
- تتبع الأخطاء: الاستثناءات غير الملتقطة ورفض الوعود تُسجل مع تتبعات المكدس.

## المراقبة

- فحص الصحة: خادم HTTP يعرض نقاط النهاية /health و /radio-health.
- إدارة الذاكرة: فحوصات دورية تطلق جمع القمامة إذا تجاوز استخدام الذاكرة العتبات.
- الإحصائيات: إحصائيات استخدام البوت (السيرفرات، الأوامر، الأذكار) تُتتبع وتُحفظ في Firebase.

## مستويات السجل

- debug: معلومات داخلية مفصلة.
- info: رسائل تشغيلية عامة.
- warn: مشاكل محتملة.
- error: أعطال حرجة.
- fatal: أخطاء توقف النظام.
