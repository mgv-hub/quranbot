# الهندسة والهيكل

## فلسفة التصميم

يتبع المشروع هندسة نمطية أحادية. يتم فصل المنطق إلى مجالات مميزة تشمل الحالة، والبيانات، والتفاعلات، والأدوات لتسهيل الصيانة. يستخدم أسماء مسارات مخصصة لإدارة هياكل الأدلة العميقة.

## تسلسل المجلدات

- core/bot/: نقطة الدخول وتهيئة العميل الرئيسية.
- core/startup/: منطق التمهيد وتسجيل الأوامر.
- core/state/: إدارة الحالة والاستمرارية.
- core/interactions/: معالجة الأزرار، والقوائم، والأوامر.
- core/data/: جلب البيانات والتخزين المؤقت.
- core/utils/: أدوات مشتركة تشمل التسجيل وFirebase.
- core/ui/: بناة التضمينات ومنشئو المكونات.
- core/package/Envira/: محمل متغيرات البيئة المخصص.

## المكونات الرئيسية

- العميل: مثيل Discord.js Client مُدار في core/bot/core.js.
- مدير الحالة: GuildStateManager و PersistentStateManager يتولان حالة وقت التشغيل وقاعدة البيانات.
- معالج التفاعلات: يوجه جميع تفاعلات Discord إلى معالجات محددة.
- محمل البيانات: يدير مكالمات API الخارجية والتخزين المؤقت للقراء والسور.

## تفاعل الوحدات

تسلسل بدء التشغيل يهيئ العميل، ويحمل البيانات، ويتصل بـ Firebase، ويستعيد الحالات السابقة. يتم توجيه التفاعلات عبر معالج مركزي يتحقق من الأذونات وفترات الانتظار قبل تنفيذ معالجات محددة.
