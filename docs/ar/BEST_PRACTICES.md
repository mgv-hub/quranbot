# أفضل الممارسات والملاحظات

## أسلوب الترميز

- النمطية: كل ميزة معزولة في ملفها الخاص.
- معالجة الأخطاء: تُستخدم كتل try-catch بشكل موسع حول العمليات غير المتزامنة.
- التسمية: المتغيرات والدوال تستخدم أسماء وصفية.
- الثوابت: قيم التكوين مركزية في configConstants.js.

## إدارة التبعيات

- Discord.js: الإصدار 14 مُستخدم لأحدث الميزات.
- Firebase: يُستخدم لاحتياجات قاعدة البيانات في الوقت الفعلي.
- الصوت: @discordjs/voice يتعامل مع اتصالات الصوت.
- مخصص: عدة مكتبات داخلية (Envira، Path Aliaser) تقلل الاعتماد الخارجي.

## القرارات الهندسية

- الحالة العامة: مُستخدمة للأداء رغم صعوبات الاختبار المحتملة.
- أولوية Firebase: الاستمرارية مُفضلة على السرعة للبيانات الحرجة.
- الاستعادة أولاً: تدفق بدء التشغيل يُفضل استعادة الحالات السابقة على التهيئة الجديدة.
