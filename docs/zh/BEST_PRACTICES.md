# 最佳实践和观察

## 编码风格

- 模块化：每个功能都隔离在自己的文件中。
- 错误处理：在异步操作周围广泛使用 try-catch 块。
- 命名：变量和函数使用描述性名称。
- 常量：配置值集中在 configConstants.js 中。

## 依赖管理

- Discord.js: 版本 14 用于最新功能。
- Firebase: 用于实时数据库需求。
- 语音：@discordjs/voice 处理音频连接。
- 自定义：几个内部库（Envira、路径别名）减少外部依赖。

## 架构决策

- 全局状态：尽管存在潜在的测试困难，但用于性能。
- Firebase 优先级：对于关键数据，持久性优先于速度。
- 恢复优先：启动流程优先恢复先前状态而不是全新初始化。
