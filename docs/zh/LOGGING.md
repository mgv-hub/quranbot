# 日志记录和监控

## 策略

- 异步队列：防止高流量期间的 I/O 阻塞。
- 归档：旧日志被压缩并存储在 storage/logs/archive 中。
- 清理：超过 60 天的日志自动删除。
- 错误跟踪：未捕获的异常和承诺拒绝与堆栈跟踪一起记录。

## 监控

- 健康检查：HTTP 服务器公开 /health 和 /radio-health 端点。
- 内存管理：如果内存使用超过阈值，定期检查触发垃圾回收。
- 统计信息：机器人使用统计信息（服务器、命令、赞念）被跟踪并保存到 Firebase。

## 日志级别

- debug: 详细的内部信息。
- info: 一般操作消息。
- warn: 潜在问题。
- error: 关键故障。
- fatal: 系统停止错误。
