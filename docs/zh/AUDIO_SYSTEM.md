# 音频系统

## 播放引擎

使用 @discordjs/voice 进行音频处理。

- 资源：从 HTTP 流创建。
- 播放器：管理播放状态和错误。
- 连接：处理语音网关通信。

## 流处理

- 验证：播放前检查 URL。
- 重试逻辑：失败的流使用备用重试。
- 持续时间：从头部或章节号估算。

## 诵读数据

- 来源：mp3quran.net 和自定义 JSON 存储库。
- 缓存：数据缓存在内存和 Firebase 中。
- 备用：如果远程源失败则使用本地文件。

## 广播系统

- 健康检查：定期监控流。
- 故障转移：自动切换到工作流。
- 分页：广播以 25 页列出。
