# ሎጊንግ እና ክትትል

## ስትራቴጂ

- አሲንክሮናስ ኪው፡ በከፍተኛ ትራፊክ ጊዜ የI/O መከልከልን ይከላከላል።
- ማከማቸት፡ አሮጌ ሎጎች ይጠቀለሉ እና በ storage/logs/archive ውስጥ ይከማቻሉ።
- ማጽዳት፡ ከ60 ቀናት በላይ የሆኑ ሎጎች በራስ ሰር ይሰረዛሉ።
- የስህተት ክትትል፡ ያልተያዙ ልዩነቶች እና ቃል ኪዳን ውድቀቶች ከስታክ ትሬስ ጋር ይመዘገባሉ።

## ክትትል

- የጤና ምርመራ፡ HTTP ሰርቨር /health እና /radio-health ነጥቦችን ያሳያል።
- የሜሞሪ አስተዳደር፡ የሜሞሪ አጠቃቀም ደረጃን ካለፈ ጊዜያዊ ፍተሻዎች ጋርቤጅ ኮሌክሽንን ያስጀምራሉ።
- ስታቲስቲክስ፡ የቦት አጠቃቀም ስታቲስቲክስ (ሰርቨሮች፣ ትዕዛዞች፣ አዝካር) ይከተባሉ እና ወደ ፋየርቤዝ ይቀመጣሉ።

## የሎግ ደረጃዎች

- debug: ዝርዝር የውስጥ መረጃ።
- info: አጠቃላይ የስራ መልእክቶች።
- warn: ሊሆኑ የሚችሉ ችግሮች።
- error: ወሳኝ ስህተቶች።
- fatal: ስርዓቱን የሚያቆሙ ስህተቶች።
